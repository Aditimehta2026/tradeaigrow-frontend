TradeAIGrow Documentation

Everything you need to set up, run, and extend the TradeAIGrow platform.

Get Started

TradeAIGrow is a full-stack trading platform built with Angular on the frontend and Node.js on the backend. Make sure you have Node.js 18+ and the Angular CLI installed before continuing. 

npm install -g @angular/cli

Clone the repository, install the dependencies for both apps, and start the frontend and API in two separate terminals. 

git clone https://github.com/your-org/tradeaigrow
cd tradeaigrow

cd tradeaigrow-frontend
npm install
ng serve

cd ../tradeaigrow-api
npm install
npm run dev

The Angular app runs at http://localhost:4200/ and the API at http://localhost:3000/ by default.

Project Structure

The repository is split into two independent applications so the frontend and backend can be built, deployed, and scaled separately.

  • tradeaigrow-frontend/: Angular 20 client application.
  • tradeaigrow-api/: Node.js / Express REST API.
  • src/app/layout: Topbar, sidebar, menu and shared layout components.
  • src/app/pages: Feature pages — dashboard, auth, trading, loan, withdrawal, etc.
  • src/app/interceptors: HTTP interceptors (auth token, error handling).
  • src/app/pages/service: Shared Angular services (auth, dashboard data, products).
Authentication

Login, registration, email verification, and password reset live under src/app/pages/auth. A JWT token is stored in localStorage and attached to every protected request by src/app/interceptors/auth.interceptor.ts. Expired or invalid tokens automatically redirect the user back to /auth/login.

Navigation & Menu

The main sidebar menu is defined in src/app/layout/component/app.menu.ts. Update the model property to add, hide, or reorder menu items.

Routing

Top-level routes are declared in src/app.routes.ts. Feature areas use lazy-loaded route files:

  • /: Public landing page.
  • /auth/*: Login, register, forgot-password, reset-password, access, error.
  • /app/dashboard: Authenticated user dashboard.
  • /app/page/*: Trading, deposit, withdrawal, loan, support, and other features.
  • /notfound: Catch-all 404 page.
Layout Service

src/app/layout/service/layout.service.ts manages layout state such as dark mode, menu mode (static / overlay), sidebar visibility, and theme color. Inject this service in any component that needs to react to layout changes.

Styling

TradeAIGrow uses Tailwind CSS utility classes for layout and spacing, with custom SCSS for app-shell styling. Global styles and CSS variables live in src/styles.scss and src/assets/layout. Prefer Tailwind utilities for new components; reach for SCSS only when a utility-only solution is awkward.

Icons & Charts

Use Feather-style icons (or the existing icon classes already wired in the project) for any new UI. For data visualization use the shared apex-chart wrapper or the chart-tab component instead of introducing a new chart library.

Notifications

All toast and alert dialogs go through SweetAlert2. Avoid mixing in other notification libraries to keep the UX consistent.

Backend API

The Express server lives in tradeaigrow-api/server.js and exposes REST endpoints under /auth, /dashboard, and feature-specific routes. Configure secrets and ports through a .env file (kept out of git via .gitignore).
